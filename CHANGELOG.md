# Changelog

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog, and this project follows Semantic
Versioning.

## [0.1.0] - 2026-05-09

Initial production-safe minimal release.

### Added

- Attribute-based encryption with `#[Encrypted]`.
- Halite-based `FieldEncryptorInterface` implementation.
- Ciphertext prefix `doctrine-encryption:halite:v1:` for detection and future
  format migration.
- Symfony bundle configuration for `key_file`, `auto_generate_key` and
  `allow_plaintext`.
- Explicit `doctrine-encryption:generate-key` and
  `doctrine-encryption:validate-key` commands.
- `--key-file` command option to override the configured key path for one-off
  operations.
- Package-specific exception hierarchy:
  `DoctrineEncryptionException`, `KeyNotFoundException`,
  `KeyGenerationException`, `InvalidCiphertextException`,
  `DecryptionFailedException` and `ConfigurationException`.
- Doctrine ORM lifecycle integration for transparent field encryption.
- Metadata cache and Doctrine proxy support.
- Readonly encrypted property validation at metadata build time.
- Unit and integration tests for key handling, plaintext policy, command
  behavior, Symfony container wiring and Doctrine ORM integration.
- `SECURITY.md`, `LICENSE`, `CHANGELOG.md`, PHPStan, PHP-CS-Fixer and CI.

### Changed

- Missing key files no longer generate a key silently by default.
- Plaintext values in encrypted fields are rejected by default.
- Internal metadata, key and subscriber classes are marked as internal API.
- Composer constraints allow PHP 8.2+, Symfony 6.4/7.4 and Doctrine ORM 2.20/3.x.

### Security

- Key files generated by the library are written with `0600` permissions.
- Key validation fails when group/world permissions are present.
- Decryption failures are wrapped in typed package exceptions.
- Plaintext values are not included in exception messages.
- The README and SECURITY docs describe key backup, plaintext migration mode,
  Docker/Kubernetes usage and lifecycle-event limitations.

### Known Limitations

- No key rotation.
- No KMS/Vault integration.
- Raw SQL, DBAL writes and bulk DQL operations are not encrypted automatically.
- Encrypted fields cannot be searched, sorted or uniquely constrained by their
  plaintext value because Halite encryption is non-deterministic.
